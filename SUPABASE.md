# Cuentas en la nube con Supabase (opcional)

El juego funciona en **local** por defecto (cada dispositivo guarda su progreso).
Si quieres **cuentas reales** (mismo progreso en varios dispositivos, sincronización
del historial), sigue estos pasos. Es **gratis**.

## 1. Crear el proyecto
1. Entra en https://supabase.com → **New project** (plan Free).
2. Apunta de **Project Settings → API**:
   - **Project URL** (algo como `https://xxxx.supabase.co`).
   - **anon public key** (la clave `anon`, es pública: segura para el front).

## 2. Crear la tabla y la seguridad (RLS)
En **SQL Editor**, pega y ejecuta esto:

```sql
create table if not exists public.profiles (
  id uuid primary key references auth.users on delete cascade,
  meta jsonb not null default '{}',
  updated_at timestamptz not null default now()
);

alter table public.profiles enable row level security;

-- Cada usuario solo puede ver/editar su propia fila
create policy "own row - select" on public.profiles
  for select using (auth.uid() = id);
create policy "own row - upsert" on public.profiles
  for insert with check (auth.uid() = id);
create policy "own row - update" on public.profiles
  for update using (auth.uid() = id);
```

### Ranking online de Glory Runs (alias + mejores tiempos)
Ejecuta también esto en el **SQL Editor** para el ranking público:

```sql
create table if not exists public.glory_runs (
  id bigint generated by default as identity primary key,
  user_id uuid references auth.users on delete cascade,
  alias text not null default 'Anónimo',
  region text not null,
  difficulty text not null,
  mode text not null,
  pools int[] not null default '{}',
  random boolean not null default false,
  duration_ms bigint not null,
  created_at timestamptz not null default now()
);

alter table public.glory_runs enable row level security;

-- Lectura pública (el ranking lo ve todo el mundo)
create policy "glory read" on public.glory_runs for select using (true);
-- Cada usuario solo inserta sus propias partidas
create policy "glory insert own" on public.glory_runs
  for insert with check (auth.uid() = user_id);

create index if not exists glory_region_diff_time
  on public.glory_runs (region, difficulty, duration_ms);
```

### Cyber PokéBall online (Cable Link: intercambios + combates fantasma)
Ejecuta también esto en el **SQL Editor** para el modo Cyber PokéBall:

```sql
-- Intercambios asíncronos: el dueño deja un Pokémon en depósito (jsonb) y otro
-- jugador lo acepta aportando el suyo. Flujo de estados:
--   open → accepted (el que acepta aporta `counter`)
--        → completed (el dueño confirma y recoge `counter`)
--        → closed (el que aceptó recoge `offered`)
--   open → cancelled (el dueño recupera su Pokémon)
create table if not exists public.cyber_trades (
  id bigint generated by default as identity primary key,
  owner_id uuid not null references auth.users on delete cascade,
  owner_alias text not null default 'Anónimo',
  gen int not null,
  offered jsonb not null,
  wanted_species int,
  status text not null default 'open',
  taker_id uuid references auth.users,
  taker_alias text,
  counter jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

alter table public.cyber_trades enable row level security;

create policy "cyber trades read" on public.cyber_trades
  for select using (true);
create policy "cyber trades insert own" on public.cyber_trades
  for insert with check (auth.uid() = owner_id);
-- El dueño gestiona su oferta (cancelar, confirmar).
create policy "cyber trades update owner" on public.cyber_trades
  for update using (auth.uid() = owner_id);
-- Cualquiera puede aceptar una oferta ABIERTA que no sea suya…
create policy "cyber trades accept" on public.cyber_trades
  for update using (status = 'open' and auth.uid() is not null and auth.uid() <> owner_id)
  with check (status = 'accepted' and taker_id = auth.uid());
-- …y el que aceptó puede cerrarla al recoger su Pokémon.
create policy "cyber trades close taker" on public.cyber_trades
  for update using (auth.uid() = taker_id);

create index if not exists cyber_trades_open
  on public.cyber_trades (status, gen, created_at desc);

-- Equipos fantasma: snapshot del equipo (máx. 3) por usuario y generación.
create table if not exists public.cyber_ghosts (
  user_id uuid not null references auth.users on delete cascade,
  gen int not null,
  alias text not null default 'Anónimo',
  team jsonb not null,
  badges int not null default 0,
  updated_at timestamptz not null default now(),
  primary key (user_id, gen)
);

alter table public.cyber_ghosts enable row level security;

create policy "cyber ghosts read" on public.cyber_ghosts for select using (true);
create policy "cyber ghosts insert own" on public.cyber_ghosts
  for insert with check (auth.uid() = user_id);
create policy "cyber ghosts update own" on public.cyber_ghosts
  for update using (auth.uid() = user_id);

-- Resultados de combates fantasma (reproducibles: misma semilla ⇒ mismo
-- resultado, cualquiera puede re-simular y verificar).
create table if not exists public.cyber_battle_results (
  id bigint generated by default as identity primary key,
  challenger_id uuid not null references auth.users on delete cascade,
  ghost_id uuid not null,
  gen int not null,
  seed bigint not null,
  challenger_won boolean not null,
  created_at timestamptz not null default now()
);

alter table public.cyber_battle_results enable row level security;

create policy "cyber results read" on public.cyber_battle_results for select using (true);
create policy "cyber results insert own" on public.cyber_battle_results
  for insert with check (auth.uid() = challenger_id);
```

> **Nota (v1, sin funciones de servidor):** el intercambio es cooperativo y se
> apoya en el cliente; un usuario malicioso podría duplicar su propio Pokémon
> editando peticiones. Asumido para la v1 (los Pokémon del modo Cyber no salen
> de ese modo). Si se endurece en el futuro: mover el canje a una Edge Function.

## 3. (Recomendado) Quitar la confirmación por email
Para que registrarse entre directo sin tener que confirmar el correo:
**Authentication → Providers → Email → desactiva "Confirm email"**.

## 4. Conectar la app
- **En local**: crea un archivo `.env` (copia de `.env.example`) con:
  ```
  VITE_SUPABASE_URL=https://xxxx.supabase.co
  VITE_SUPABASE_ANON_KEY=eyJ... (anon key)
  ```
- **En el deploy (GitHub Pages)**: en el repo, **Settings → Secrets and variables →
  Actions → Variables**, crea:
  - `SUPABASE_URL` = la Project URL
  - `SUPABASE_ANON_KEY` = la anon key
  (El workflow ya las usa al construir.)

## Listo
Al volver a desplegar, aparecerá **Ajustes → ☁️ Cuenta** para registrarse/iniciar
sesión. El historial (récords, Pokédex, Glory Runs) se sincroniza al terminar cada
partida. Si las claves están vacías, todo sigue funcionando en local.
